ソースファイル分割

C

コードを分割する

includeを用いる

includeで<>で囲むとライブラリを読み込むが、""(ダブルクォーテーション)で囲むとそのファイルを同一ディレクトリから読み込む

そして、読み込んだ内容を展開する

例

#include <stdio.h>
#include "list.c"

ヘッダファイル

マクロや定数の定義、関数のプロトタイプ宣言を書いたヘッダファイルを取り込むこと多い

ヘッダを利用することで宣言と実装を分けている

#ifndef ~ #define ~ #endifで二重インクルード(別々の箇所で同じファイルが複数回インクルードされること)を防止しています"

foo.h
#ifndef _FOO_H_
#define _FOO_H_

#include <stdio.h>
// 定数の定義
#define SIZE 10
// マクロの定義
#define max(a, b) ((a) > (b)) ? (a) : (b)
// 関数のプロトタイプ宣言
int bar(int);

#endif
foo.c
#include "foo.h"
int n = 20;
// 関数の実装
int bar(int a) {
    return max(a, SIZE);
}
main.c
#include <stdio.h>
#include "foo.h"

extern int n;   // externは他のファイルで宣言されていることを示す
int main(void) {
    printf("%d\n", bar(n));
    return 0;
}
コンパイル
gcc main.c foo.c
